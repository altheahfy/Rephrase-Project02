Rephraseプロジェクト：JSON入力仕様マニュアル（最新版）

【目的】

このドキュメントは、ChatGPTがRephrase構文データベースに対応するJSONファイルを正しく生成するための仕様書である。今後、出力形式に変更が生じた場合、本ドキュメントを随時更新すること。

【基本構造】

JSONは1文例に対して一つのオブジェクトを表現す。

{
  "構文ID": 5001,
  "例文ID": "ex001",
  "V_group_key": "figure out",
  "V_display": "figured out",
  "原文": "At the very last second, the thoughtful woman must have figured out that he wanted to avoid confrontation because he feared upsetting her.",
  "slot_data": [
    ...
  ]
}

【必須項目】

キー

内容

備考

構文ID

文法項目のID (数値)

同一構文の例文は同一IDを持つ

例文ID

個別例文の識別子

"ex001"のように、ex+通し番号の形式

V_group_key

構文代表動詞（句動詞可）

"give"や"figure out"など、構文テンプレートの基準になる

V_display

実際の文中の動詞表現

"figured out"や"gave"など、動詞の形を含む

原文

英文の整文

ビジュアルライズされた検証用の基本文

【slot_dataの構造】

slot_dataは各slotの情報を配列として持つ

各slotは下記情報を持つ

{
  "slot": "M1",
  "phrase": "At the very last second",
  "type": "word",
  "slot_text": "後の日程で、最終的に"
}

clause型slotの場合

{
  "slot": "O1",
  "phrase": "that he wanted to avoid confrontation",
  "type": "clause",
  "subslots": [
    { "id": "S", "phrase": "he", "subslot_text": "指示語【その】" },
    { "id": "V", "phrase": "wanted to avoid confrontation", "subslot_text": "" }
  ]
}

【作成方針】

ChatGPTは以下の順番でJSONを生成すこと

文法項目に基づく基本例文を考案

V_group_key / V_display / 例文ID / 原文を満たすようメタ情報を手動で入力

文を構文slotに分解し、slot_dataを構築

clause型にはsubslotを正しく含める

【注意事項】

未来的に、追加項目や入れ替えが発生した場合は、本マニュアルをChatGPT側で通知の上、更新すること

更新が経過した場合は、既存JSONデータへの逆換処理が必要になる場合がある



【clause型 subslot の分解ルール補足】
clause型の sub-s には、主語の先行詞に付随する限定詞や関係詞を含めること。
例：
- the woman who seemed indecisive → sub-s は "the woman who"
- the boy that was late → sub-s は "the boy that"
このようにすることで、限定詞や関係詞が上位slot・subslotのどこにも表示されない事態を防ぐ。
また、clause型 subslot の分解は上位slotの分解ルールと同等の基準に従うこと。
例：
- sub-s（関係詞＋主語を含めた主語部分。例: that he）
- sub-v（動詞部分。例: wanted）
- sub-o1（目的語部分。例: to avoid confrontation）
- sub-m1（副詞要素。例: because）


【subslot 名の指定ルール】
subslot の slot 名は以下のいずれかを必ず使用すること。
sub-s, sub-v, sub-aux, sub-o1, sub-o2, sub-c1, sub-c2, sub-m1, sub-m2, sub-m3

【slot_text の付与ルール】
slot_text は type が "word" の場合に付与する。句・節の場合は slot_text を付与しない。
また、slot_text の内容は補助テキストルール（別ファイル slottext.json）に従うこと。

【phrase 表記統一ルール】
phrase の記載は例文原文に準拠し、不要な大文字化・小文字化・ピリオドの追加は行わない。
